Fragmentは、複数のQueryやMutationなどで利用される共通部分をまとめることができる機能。

例えば、スキーマ定義言語側が以下のように定義されている場合を考える。

type Person
{
    name: String!
    age: Int
    weight: Int
    height: Int
    country: String
}

type Query
{
    persons: [Person]
}

このとき、複数のQueryで利用するFieldが共通している場合、以下のようなFragmentを用意するとよい。

fragment PersonFragment on Person
{
    name
    country
}

QueryでこのFragmentを利用する場合、次のように記述する。

query
{
    persons
    {
        ...PersonFragment
    }
}
